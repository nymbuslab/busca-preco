Guia passo a passo para instalar a API no servidor do cliente e conectar ao site público.

Este manual te leva do ZIP recebido até o site rodando em produção. Não é necessário conhecimento técnico avançado, mas você precisa de acesso de administrador no servidor onde o ERP GR7 está instalado e de uma conta na Cloudflare com o domínio do cliente cadastrado.

Como funciona

A API roda localmente no servidor do cliente (porque o banco MySQL é local). Um pequeno programa chamado Cloudflare Tunnel expõe essa API local para a internet num endereço HTTPS público (ex: https://api.empresa.com.br). O site no Vercel chama esse endereço HTTPS público, que é roteado de volta pra API local.

Operador (qualquer dispositivo, qualquer lugar) │ │ HTTPS ▼ busca-preco-plum.vercel.app (Front) │ │ HTTPS ▼ api.<empresa>.com.br (Cloudflare gerencia o cert) │ │ tunnel ▼ cloudflared (rodando no servidor do cliente) │ │ HTTP localhost ▼ BuscaPrecoAPI.exe :8000 │ ▼ MySQL local

Os 2 programas (BuscaPrecoAPI.exe e cloudflared.exe) ficam rodando como Serviços do Windows — sobem sozinhos no boot da máquina, sem precisar abrir terminal ou logar.

Importante

Se o cliente não tem domínio na Cloudflare ainda, providencie antes. É grátis se já existe domínio em outro registrador — basta trocar os nameservers para os da Cloudflare (o painel da Cloudflare te orienta).

Passo a passo

01

Extrair o pacote

Você recebeu o arquivo pacote-BuscaPreco.zip. Extraia ele dentro da pasta do ERP — assim os dois ficam vizinhos.

Exemplo de estrutura final:

D:\GR7\GR7\
├── config.ini             ← já existe (do ERP)
├── BuscaPreco\            ← extraído do ZIP
│   ├── BuscaPrecoAPI\
│   ├── cloudflared.exe
│   ├── nssm.exe
│   ├── tunnel\
│   ├── instalar.bat
│   ├── desinstalar.bat
│   └── LEIA-ME.txt
└── ... (resto do ERP)
Por que aqui?

A API lê automaticamente o config.ini que o ERP mantém. Colocando o pacote na pasta vizinha, ela acha as configurações de banco sozinha. Não copie o config.ini — deixe-o onde está.

02

Configurar o Cloudflare Tunnel

Esta etapa autentica o servidor do cliente na Cloudflare e cria a "ponte" pública que o Vercel vai usar. Só precisa fazer uma vez por cliente.

Abra o PowerShell como Administrador (botão direito no ícone do PowerShell → "Executar como administrador") e navegue até a pasta do pacote:

PS> cd D:\GR7\GR7\BuscaPreco

2.1 — Autenticar com a conta Cloudflare

PS> .\cloudflared.exe tunnel login

Vai abrir o navegador. Faça login na Cloudflare, selecione o domínio do cliente e clique em Authorize. Quando voltar pro terminal, a mensagem deve ser You have successfully logged in..

2.2 — Criar o túnel

PS> .\cloudflared.exe tunnel create busca-preco-<nome-cliente>

Troque <nome-cliente> por uma identificação curta (ex: busca-preco-mercado-x). O comando imprime:

Created tunnel busca-preco-mercado-x with id abc12345-de67-89ab-cdef-1234567890ab

Anote esse tunnel-id (UUID). Você vai precisar dele já já. O comando também criou um arquivo de credenciais em:

C:\Users\<seu-usuario>\.cloudflared\<tunnel-id>.json

2.3 — Preencher o config.yml

Dentro da pasta do pacote, abra tunnel\config.yml.exemplo em um editor (Bloco de Notas serve). Você verá três placeholders pra trocar:

tunnel: <TUNNEL-ID>
credentials-file: <CAMINHO-DO-JSON>

ingress:
  - hostname: <SUBDOMINIO-DO-CLIENTE>
    service: http://localhost:8000
  - service: http_status:404

Exemplo preenchido (não copie literal — use os seus valores):

tunnel: abc12345-de67-89ab-cdef-1234567890ab
credentials-file: C:\Users\administrador\.cloudflared\abc12345-de67-89ab-cdef-1234567890ab.json

ingress:
  - hostname: api.mercado-x.com.br
    service: http://localhost:8000
  - service: http_status:404

Depois, renomeie o arquivo de config.yml.exemplo para config.yml (remove o .exemplo). No PowerShell:

PS> Move-Item .\tunnel\config.yml.exemplo .\tunnel\config.yml

2.4 — Apontar o DNS

PS> .\cloudflared.exe tunnel route dns busca-preco-<nome-cliente> api.<empresa>.com.br

Use o mesmo nome do túnel do passo 2.2 e o mesmo subdomínio que você colocou no hostname. Isso cria automaticamente um registro CNAME na zona DNS da Cloudflare.

Pronto

O túnel está configurado. Próximo passo é registrar os serviços do Windows.

03

Rodar o instalador

Ainda na pasta do pacote, no PowerShell admin (ou clicando direito → "Executar como administrador"):

PS> .\instalar.bat

O script vai:

  • Registrar o serviço BuscaPrecoAPI (a API empacotada)
  • Registrar o serviço BuscaPrecoTunnel (a ponte Cloudflare)
  • Configurar os 2 pra subir automaticamente no boot do Windows
  • Iniciar os 2 agora

Saída esperada:

============================================================
  Instalador BuscaPreco
============================================================

 [1/2] Registrando servico BuscaPrecoAPI...
      OK
 [2/2] Registrando servico BuscaPrecoTunnel...
      OK

 Iniciando servicos...

============================================================
  Instalacao concluida
============================================================
Se aparecer este aviso

Servico BuscaPrecoTunnel NAO instalado. Motivo: tunnel\config.yml nao encontrado. — é porque você esqueceu de renomear o arquivo no passo 2.3 (tirar o .exemplo). Renomeie e rode instalar.bat de novo.

04

Verificar se está funcionando

(a) Os 2 serviços estão rodando?

PS> sc query BuscaPrecoAPI
PS> sc query BuscaPrecoTunnel

Ambos devem mostrar STATE : 4 RUNNING.

(b) A API responde localmente?

PS> curl http://localhost:8000/docs

Deve retornar HTML (página do Swagger). Status 200.

(c) A API responde via tunnel HTTPS? Abre no navegador (de qualquer dispositivo, pode ser celular no 4G):

https://api.<empresa>.com.br/docs

Deve abrir a mesma página Swagger. Se aparecer, está tudo conectado.

(d) Uma busca de verdade. Use o endereço acima trocando /docs pelo endpoint:

https://api.<empresa>.com.br/produtos/barras/<codigo>

Substitua <codigo> por um código de barras real do banco do cliente. Deve retornar JSON com os dados do produto.

Tudo verde?

Avance pro passo 5. Se algum dos 4 testes acima falhou, vá direto pra seção Solução de problemas.

05

Atualizar a variável de ambiente no Vercel

O site precisa saber onde a API está. Acesse vercel.com e vá até o projeto busca-preco-plum:

  1. Menu Settings → Environment Variables.
  2. Se já existir VITE_API_URL, clique nele e edite. Se não, crie:
    • Key: VITE_API_URL
    • Value: https://api.<empresa>.com.br (o subdomínio do passo 2.3)
    • Environments: marque Production, Preview e Development.
  3. Clique em Save.
  4. Vá em Deployments, último deploy → menu ⋯ → Redeploy.
    • Desmarque a opção "Use existing Build Cache" — força o site a pegar o novo endereço.
  5. Aguarde uns 30 segundos a 1 minuto. Quando o status ficar Ready, abre https://busca-preco-plum.vercel.app e testa uma busca.
Implantação completa

O cliente já pode consultar preços de qualquer lugar. O servidor reinicia → os serviços sobem sozinhos → continua funcionando.

Solução de problemas

O serviço BuscaPrecoAPI não inicia

Veja o log:

PS> Get-Content .\logs\api.log -Tail 50

Causas mais comuns:

  • MySQL não está rodando — o ERP está aberto? Se não, suba o MySQL primeiro.
  • Credenciais erradas no config.ini — confira Servidor, Base_Dados, Caminho_Servidor, MySQL_Porta na seção [Configuracoes].
  • Porta 8000 já está ocupada — netstat -ano | findstr 8000 e mate o processo.

O serviço BuscaPrecoTunnel não conecta

PS> Get-Content .\logs\tunnel.log -Tail 50
  • O caminho em credentials-file dentro do config.yml precisa ser o caminho absoluto correto do arquivo .json. Se o usuário Windows for diferente, o caminho está errado.
  • O subdomínio precisa estar apontado via tunnel route dns. Verifique com: 
    PS> .\cloudflared.exe tunnel info busca-preco-<nome-cliente>

Erro "Failed to load Python DLL" ao tentar o exe

A pasta BuscaPrecoAPI\_internal\ está incompleta. Apague a pasta do BuscaPrecoAPI inteira e recopie de dentro do ZIP original. Pode ter sido bloqueada pelo Windows Defender — adicione a pasta como exceção no antivírus se persistir.

O site mostra "Produto não encontrado" pra códigos que existem

  1. Abra o DevTools do navegador (F12) → aba Network → faça uma busca → clica na requisição falha.
  2. Se for erro de CORS, confira que a URL do Vercel está na lista de origens permitidas no srv-buscapreco.py (caso seja URL nova).
  3. Se for erro 502/504, o tunnel está caído — confira o serviço.
  4. Se a resposta vier {"exatos":[],"similares":[]} com status 200, a API funcionou mas o código não existe no banco do cliente.

Logs ficam grandes demais

O NSSM já rotaciona em 10MB. Se quiser limpar:

PS> Remove-Item .\logs\*.log

Operação do dia a dia

Reiniciar um serviço manualmente

PS> sc stop  BuscaPrecoAPI
PS> sc start BuscaPrecoAPI

Atualizar a versão da API

  1. Pare os serviços: sc stop BuscaPrecoAPI.
  2. Substitua a pasta BuscaPrecoAPI\ pelo novo conteúdo do ZIP. Não mexa em cloudflared.exe, nssm.exe, tunnel\config.yml nem config.ini.
  3. Suba os serviços: sc start BuscaPrecoAPI.

Desinstalar tudo

PS> .\desinstalar.bat

Para e remove os 2 serviços. Os arquivos da pasta ficam — apague manualmente se quiser.
